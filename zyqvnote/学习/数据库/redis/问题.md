```
问题1：jedis怎么控制主节点的写，从节点的读      
     答：非集群环境下通过哨兵；集群环境下通过客户端代码判断是否是主节点（book），从节点读取时，会路由到key对应的主节点
问题2：redis集群如何实现负载均衡                      
     答：通过映射算法，和nginx iphash差不多
问题3：redis sentinel实现故障转移后，出现故障的节点的ip会变化吗，成为主节点的从节点的ip会变化吗 
     答：不会变化，只是地位变了，数据需要备份
问题4：redis hset?
     答：在数据量小的时候是ziplist作为实现，数据量大的时候用hashtable作为实现
```

#### redis阻塞的内部原因
```
1. api或者数据结构使用不合理。例如使用了hgetall这样的命令，或者大对象。
2. AOF或者RDB持久化阻塞
3. cpu饱和。可能过度优化内存或者使用了高算法复杂度的命令
```
